Part 1 · 0 – 15 min

Markdown 是什麼？為什麼你需要學它？

Markdown 是一種輕量標記語言——用極少的特殊符號，讓純文字變得有結構。你不需要 Word，不需要 HTML，只要一個文字編輯器。

你輸入的 .md 檔→渲染後的樣子
# 魯迅 **原名**周樹人，浙江紹興人。 > 橫眉冷對千夫指， > 俯首甘為孺子牛。 - 《呐喊》1923 - 《彷徨》1926

Part 3 · 45 – 75 min

從 Markdown 到 AGENTS.md：用文字指揮 AI

現在把剛學的語法用在真正的目的上。AGENTS.md 就是一份 Markdown 文件——但它的讀者不是人，是 AI coding agent。你在這份文件裡寫的每一行，都會在 AI 開始工作之前被讀取，成為它的「工作守則」。

結構即指令：# 角色設定 這個標題，讓 AI 知道「這個區塊是在定義我的身份」。## 禁止行為 這個標題，讓 AI 知道「這個區塊是邊界規則」。Markdown 的結構，直接對應 AI 的認知區塊。

一份 AGENTS.md 的標準結構

典型 AGENTS.md 的骨架（Markdown 語法 + 說明）
# 專案名稱與概述 ← H1：給 AI 定錨（這是什麼專案） ## 角色與身份 ← H2：這個 AI 扮演什麼角色 你是一名台灣文學研究助理... ## 可以使用的工具 ← H2：能力範圍 - 搜尋學術資料庫 - 讀取 .txt 和 .md 檔案 - **不可以** 修改原始語料 ## 工作流程 ← H2：思考與行動的順序 1. 先閱讀使用者的問題 2. 確認需要哪些資料 3. 回應前列出你的推理步驟 ## 輸出格式 ← H2：回應的形式要求 - 用繁體中文回答 - 引用文本時用 `>` 引用格式 - 每段分析不超過 200 字 ## 限制與邊界 ← H2：不該做的事 > **絕對不可以** 捏造不存在的引文。 > 不確定時說「我不確定」，不要猜測。

進一步：system prompt vs AGENTS.md

System Prompt（聊天介面用）自由格式
你是一名台灣文學研究員， 專長是日治時期的文學分析。 請用繁體中文回答， 不要捏造引文。
AGENTS.md（程式 / 工具用）結構化 Markdown
# 台灣文學研究助理 ## 角色 台灣文學研究員，專長日治時期。 ## 輸出格式 - 語言：繁體中文 ## 限制 > 不得捏造引文。
重要差異：System prompt 是「對話層」的指令，只在一次對話裡有效。AGENTS.md 是「專案層」的指令，會被 Cursor、Claude Code、Codex 等工具在每次工作開始前自動讀取——它的效力更持久，也更系統性。
